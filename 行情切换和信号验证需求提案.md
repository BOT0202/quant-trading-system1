## 🎯 需求概述

### 需求1: 行情数据源可切换
**背景**: 需要支持多数据源切换，先接入币安XAUUSDT，后期可切换至其他数据源（如盈透、CME外盘期货行情）

### 需求2: 信号详情页面行情展示
**背景**: 在信号详情页面需要展示当前信号品种的实时行情数据，用于人工校验信号计算结果的正确性

## 📋 详细需求规格

### 需求1: 行情数据源可切换

#### 1.1 数据源架构需求
- **多数据源支持**: 系统需要支持同时接入多个行情数据源
- **数据源切换**: 用户可以手动或自动切换数据源
- **数据源管理**: 管理员可以配置和管理数据源
- **数据源监控**: 监控各数据源的状态和质量

#### 1.2 币安数据源接入 (第一阶段)
- **交易品种**: XAUUSDT (币安黄金永续合约)
- **数据接口**: 币安WebSocket API + REST API
- **数据范围**: 
  - 实时行情数据
  - K线数据 (1m, 5m, 15m, 1h, 4h, 1d)
  - 深度数据 (可选)
  - 成交数据 (可选)
- **更新频率**: 实时更新，延迟<100ms

#### 1.3 未来数据源规划
- **盈透证券 (IBKR)**: CME黄金期货行情
- **其他交易所**: 根据需要接入更多数据源
- **数据源特性**:
  - 支持不同数据格式和协议
  - 支持不同数据更新频率
  - 支持不同数据质量要求

#### 1.4 切换功能需求
- **手动切换**: 用户可以在界面上手动选择数据源
- **自动切换**: 系统可以根据规则自动切换数据源
- **切换条件**:
  - 数据源故障或延迟过高
  - 数据质量不达标
  - 用户偏好设置
- **切换效果**: 无缝切换，不影响用户使用

#### 1.5 数据一致性需求
- **数据标准化**: 不同数据源的数据需要标准化处理
- **时间同步**: 确保不同数据源的时间同步
- **价格校准**: 不同数据源的价格差异处理
- **历史数据**: 切换数据源时历史数据的处理

### 需求2: 信号详情页面行情展示

#### 2.1 行情展示区域设计
- **位置**: 信号详情页面的显著位置
- **布局**: 与信号分析区域相邻或集成
- **大小**: 足够展示关键行情信息
- **交互**: 支持展开/收起，调整显示内容

#### 2.2 展示内容需求
- **基础行情信息**:
  - 当前价格 (最新价)
  - 涨跌幅和涨跌额
  - 24小时最高/最低价
  - 24小时成交量
  - 买卖盘口 (买一/卖一)

- **K线图表展示**:
  - 实时K线图表
  - 多时间周期切换 (1m, 5m, 15m, 1h, 4h, 1d)
  - 技术指标叠加 (MA, MACD, RSI等)
  - 信号标记点 (显示信号生成位置)

- **深度数据展示** (可选):
  - 买卖盘深度图
  - 大单成交记录
  - 资金流向分析

#### 2.3 校验功能需求
- **信号位置标记**: 在K线图上标记信号生成位置
- **价格对比**: 显示信号生成时的价格和当前价格
- **技术指标对比**: 显示信号生成时的技术指标值和当前值
- **校验结果**: 提供简单的校验结果提示 (如: 信号有效/需复核)

#### 2.4 交互功能需求
- **时间轴控制**: 可以查看历史时刻的行情
- **数据刷新**: 手动刷新和自动刷新选项
- **图表操作**: 缩放、平移、指标切换
- **数据导出**: 导出当前显示的行情数据

#### 2.5 用户体验需求
- **响应速度**: 行情数据加载<1秒
- **更新频率**: 实时更新，可配置更新间隔
- **错误处理**: 行情数据获取失败时的友好提示
- **帮助信息**: 提供行情数据的解释和说明
