## 🔧 系统工作原理

### 活动监控流程：
```
角色工作 → 记录活动 → 存储日志 → 每天检查 → 3天无活动 → 自动重置
```

### 重置触发条件：
- **时间阈值**: 连续3天（72小时）无活动记录
- **活动定义**: 任何工作产出、消息发送、任务更新
- **检查频率**: 每天9:00自动检查

### 重置操作内容：
1. **清理临时文件** - 删除3天前的临时文件
2. **发送重置通知** - 提醒角色重新激活
3. **更新项目看板** - 状态改为"已重置"
4. **进度清零** - 任务进度重置为0%
5. **记录重置日志** - 详细记录重置操作
