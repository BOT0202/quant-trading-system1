```python
import psycopg2
import pytest

def test_database_connection():
    """测试数据库连接"""

        host="localhost",
        port=5433,
        database="quant_test",
        user="test_user",
        password="test_password123"

def test_table_exists():
    """测试表是否存在"""
    conn = get_test_connection()
    cursor = conn.cursor()
    
    cursor.execute("""
    SELECT COUNT(*) 
    FROM information_schema.tables 
    WHERE table_name = 'test_strategies'
    """)
    
    count = cursor.fetchone()[0]
    assert count == 1
```

## 🚨 故障排除

### 常见问题1: Docker启动失败
```bash
# 检查Docker服务状态
sudo systemctl status docker

# 查看详细错误日志
docker-compose -f docker-compose.test.yml logs --tail=50

# 清理旧容器
docker-compose -f docker-compose.test.yml down -v
docker system prune -a

# 重新启动
docker-compose -f docker-compose.test.yml up -d
```

### 常见问题2: 数据库连接失败
```bash
# 检查数据库服务
docker exec -it quant-test-postgres pg_isready -U test_user -d quant_test

# 查看数据库日志
docker logs quant-test-postgres

# 重启数据库服务
docker-compose -f docker-compose.test.yml restart test-postgres
```

### 常见问题3: 端口冲突
```bash
# 查看端口占用
sudo netstat -tulpn | grep :8001
sudo netstat -tulpn | grep :5433

# 修改端口配置
# 编辑docker-compose.test.yml，修改ports配置
# 例如: "8002:8000" 改为 "8001:8000"
```

### 常见问题4: 内存不足
```bash
# 查看内存使用
docker stats

# 限制容器内存
# 在docker-compose.test.yml中添加:
# test-app:
#   deploy:
#     resources:
#       limits:
#         memory: 1G

# 清理无用资源
docker system prune -a --volumes
```

## 🔄 持续集成

### GitHub Actions配置
```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_DB: quant_test
          POSTGRES_USER: test_user
          POSTGRES_PASSWORD: test_password123
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
      
      redis:
        image: redis:7
        options: >-
          --health-cmd "redis-cli ping"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 6379:6379
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.11'
    
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install pytest pytest-cov
    
    - name: Run tests
      env:
        DATABASE_URL: postgresql://test_user:test_password123@localhost:5432/quant_test
        REDIS_URL: redis://localhost:6379/0
      run: |
        python -m pytest --cov=./ --cov-report=xml --cov-report=html
    
    - name: Upload coverage
      uses: codecov/codecov-action@v3
```

## 📚 测试策略

### 测试金字塔
```
        E2E测试 (10%)
          /\
         /  \
        /    \
集成测试 (20%)
      /\
     /  \
    /    \
单元测试 (70%)
```

### 测试类型
1. **单元测试**: 测试单个函数或类
2. **集成测试**: 测试模块间集成
3. **API测试**: 测试REST API接口
4. **性能测试**: 测试系统性能
5. **安全测试**: 测试安全漏洞
6. **兼容性测试**: 测试不同环境兼容性

### 测试优先级
- **P0**: 核心功能测试（必须通过）
- **P1**: 重要功能测试（应该通过）
- **P2**: 边缘情况测试（最好通过）
- **P3**: 性能优化测试（可选）

## 🎯 最佳实践

### 测试代码规范
1. **命名规范**: 测试函数以`test_`开头
2. **独立性**: 测试之间相互独立
3. **可重复性**: 测试结果可重复
4. **快速反馈**: 测试运行速度快

### 测试数据管理
1. **使用fixtures**: 避免硬编码测试数据
2. **数据隔离**: 测试数据相互隔离
3. **清理机制**: 测试后清理数据
4. **数据工厂**: 使用工厂模式生成数据

### 测试环境管理
1. **环境隔离**: 测试环境与生产环境隔离
2. **配置管理**: 环境配置版本控制
3. **资源管理**: 合理分配测试资源
4. **监控告警**: 测试环境监控告警
