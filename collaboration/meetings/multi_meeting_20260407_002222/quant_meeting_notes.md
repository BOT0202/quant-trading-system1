## 🎯 项目概述
**项目目标**: 搭建一套基于多种策略的量化交易信号系统，为交易员提供外盘黄金的交易信号辅助决策。

**核心功能**:
1. 多策略信号生成与整合
2. 实时行情数据接入
3. 信号推送与通知
4. 策略回测与优化
5. 风险控制与管理

**交易品种**: 外盘黄金（XAU/USD等）

## 📋 会议讨论记录

### 第一阶段：项目背景与目标

**主持人(YangCheng)**: 大家好！今天我们启动外盘黄金量化交易信号系统项目。这个系统的主要目标是为交易员提供基于多种策略的交易信号，辅助他们进行外盘黄金交易。请各角色确认对职责的理解。

**产品经理(Alex)**: 明白！我将从交易员需求调研开始，重点了解他们目前的手动交易痛点，需要哪些类型的信号（趋势、反转、震荡等），以及期望的信号推送方式（APP推送、邮件、短信等）。

**技术负责人(Brian)**: 收到。我需要了解几个关键技术点：1) 需要接入哪些行情数据源？2) 信号计算频率（秒级、分钟级）？3) 系统需要支持多少并发用户？4) 数据存储和回测的历史数据量预估？

**运营专家(Cathy)**: 理解。我将重点收集和测试黄金交易策略，包括技术指标策略、基本面策略、市场情绪策略等。需要明确策略回测的时间周期和评估指标（胜率、盈亏比、夏普比率等）。

**测试专家(Taylor)**: 明白。我会从交易员的使用场景出发，测试信号的准确性、及时性，以及系统的易用性。特别关注在极端行情下的系统表现和风险提示。

### 第二阶段：关键需求讨论

**产品经理(Alex)**: 基于初步调研，我建议系统包含以下核心功能：
1. **信号看板**: 实时显示各策略生成的交易信号
2. **策略管理**: 策略启用/禁用、参数调整
3. **回测平台**: 历史数据回测策略效果
4. **风险监控**: 仓位、止损、波动率监控
5. **通知中心**: 多渠道信号推送

**技术负责人(Brian)**: 从技术架构角度，我建议：
1. **数据层**: 使用时序数据库存储行情数据，关系数据库存储用户和策略数据
2. **计算层**: 微服务架构，策略计算服务独立部署
3. **推送层**: WebSocket实时推送 + 消息队列异步通知
4. **回测引擎**: 支持并行回测，提高效率

**运营专家(Cathy)**: 对于策略方面，我建议：
1. **初期策略**: 从简单的技术指标策略开始（MA、MACD、RSI等）
2. **数据需求**: 需要至少5年的历史黄金数据用于回测
3. **评估标准**: 胜率>55%，盈亏比>1.5，最大回撤<20%
4. **优化流程**: 定期回顾策略表现，动态调整参数

**测试专家(Taylor)**: 从测试角度，我需要：
1. **测试环境**: 模拟交易环境，避免实盘风险
2. **测试数据**: 包含正常行情和极端行情数据
3. **用户体验**: 信号展示清晰，操作简便，响应快速
4. **风险测试**: 系统故障时的降级方案和恢复机制
