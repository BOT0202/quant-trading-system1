# 多角色协作系统 - 产品、技术、运营

## 系统概述

这是一个模拟产品、技术、运营三个角色协作的框架。每个角色有特定的职责和视角，通过消息传递和任务分配进行协作。

## 角色定义

### 1. 产品经理 (Product Manager)
**职责：**
- 市场调研和用户需求分析
- 产品规划和功能设计
- 需求文档撰写
- 产品路线图制定
- 与技术和运营团队协调

**核心能力：**
- 用户思维
- 数据分析
- 需求优先级排序
- 原型设计
- 项目管理

### 2. 技术负责人 (Tech Lead)
**职责：**
- 技术架构设计
- 开发计划制定
- 代码质量把控
- 技术风险评估
- 团队技术指导

**核心能力：**
- 系统设计
- 编程能力
- 技术选型
- 性能优化
- 问题排查

### 3. 运营专家 (Operations Specialist)
**职责：**
- 用户增长策略
- 内容运营
- 数据分析与监控
- 用户反馈收集
- 活动策划执行

**核心能力：**
- 数据分析
- 内容创作
- 用户沟通
- 活动策划
- 渠道管理

## 协作流程

### 1. 项目启动阶段
```
产品 → 技术：提出产品需求和技术可行性讨论
产品 → 运营：讨论市场定位和用户获取策略
技术 → 运营：了解技术实现对运营的影响
```

### 2. 开发执行阶段
```
技术 → 产品：反馈开发进度和技术难点
运营 → 产品：提供用户反馈和市场数据
产品 → 运营：同步产品功能和发布时间
```

### 3. 上线运营阶段
```
运营 → 技术：报告线上问题和用户反馈
技术 → 运营：提供数据接口和分析工具
产品 → 技术：收集用户行为数据需求
```

## 消息格式

```json
{
  "from": "角色名称",
  "to": "角色名称",
  "type": "请求/回复/通知/问题",
  "topic": "主题",
  "content": "具体内容",
  "priority": "高/中/低",
  "deadline": "截止时间（可选）",
  "attachments": ["相关文件或数据"]
}
```

## 协作工具

1. **需求池** - 产品维护的需求列表
2. **技术任务板** - 技术团队的任务跟踪
3. **运营日历** - 运营活动计划表
4. **数据看板** - 三方共享的数据仪表盘
5. **会议纪要** - 协作会议记录

## 启动脚本

```bash
# 启动三个角色的协作会话
./start_collaboration.sh
```

## 示例场景

### 场景1：新功能开发
1. 产品提出新功能需求
2. 技术评估实现方案
3. 运营规划上线推广
4. 三方协调时间节点

### 场景2：问题处理
1. 运营发现用户问题
2. 技术排查问题原因
3. 产品决定解决方案
4. 运营通知用户处理结果

### 场景3：数据分析
1. 产品提出分析需求
2. 技术提供数据支持
3. 运营执行分析报告
4. 三方讨论优化方案
